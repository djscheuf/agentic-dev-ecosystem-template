# Audit Current System State

**Inputs**: One [[../artifacts/User Story]], One [[../artifacts/Current System State]]  
**Outputs**: One [[../artifacts/Current State Audit]]

**Description**: Quick review of existing implemented stories, ADRs, test cases, and code to understand the baseline from which the story will build. This information typically lives in developer knowledge but should be verified against documentation.

**Process**:
1. Review existing implemented stories
2. Review existing ADRs
3. Review existing test cases
4. Review relevant code sections
5. Document current reality for this story's scope
